ARQUITECTURA

Base seria para vender a muchos clubes sin mezclar datos.

Backend .NET por capas, Entity Framework Core Code First, JWT, refresh tokens, roles y aislamiento por tenant usando el header X-Tenant-Slug.

Frontend estaticoVanilla JS, hash routing, API client centralizado, Netlify-ready.
API .NETControllers, middlewares, Swagger, Serilog, CORS y manejo global de errores.
ApplicationServicios, DTOs, reglas de reservas, cupos, pagos y membresias.
InfrastructureSQL Server, EF Core, migrations, seed, repositorios y Unit of Work.
Tenant isolationCada query relevante se filtra por ClubTenantId.
Reglas configurablesLimites semanales, costo de invitados, horarios y pagos por club.
Preparado para deployBackend Somee, frontend Netlify y SQL Server.
EJEMPLOS DE USO

El sistema cubre la operacion real, no solo la demo visual.

Socio reserva doblesElige cancha y horario, carga tres jugadores, marca quienes son invitados y ve el total a cobrar.
Admin registra pagoSelecciona motivo: membresia, invitados de cancha, reserva u otro. El reporte queda separado.
Profesor reserva cupoDesde sus clases, busca socios elegibles y reserva cupos sin acceder a configuracion del club.
Club cambia reglasActualiza limites semanales, dias de pago, lista de espera, costo de invitados e idioma.
Pago vencido bloquea reservaSegun configuracion, el socio con deuda queda bloqueado o advertido al reservar.
SuperAdmin escala SaaSPuede administrar clubes, activar tenants y mirar metricas generales de uso.
